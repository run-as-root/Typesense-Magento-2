# Conversational Search (RAG) Design

**Date:** 2026-03-24
**Module:** `run-as-root/magento2-typesense`

## Goal

Add natural language / conversational search to the Typesense module. Customers can ask questions like "what's a warm jacket for hiking under $100?" and get an AI-generated answer above the product grid, powered by Typesense's built-in RAG.

## Prerequisites

- Typesense server v30+ (upgrade from v27.1)
- `typesense/typesense-php` SDK v6.0+ (upgrade from v4.9)
- OpenAI API key (for LLM responses)

## Decisions

- **LLM Provider:** OpenAI only (simplest, widest adoption)
- **Frontend UX:** Enhance existing instant search page — AI answer box above product grid
- **Embeddings:** Auto-generated by Typesense using built-in model (`ts/all-MiniLM-L12-v2`), source fields configurable in admin
- **Version strategy:** Bump minimum Typesense requirement to v30+

## Architecture

### Data Flow

```
Customer types natural language query
    -> Frontend JS sends to Typesense multi_search with conversation=true
    -> Typesense performs semantic search on product embeddings
    -> Top results sent as context to OpenAI via Typesense's built-in RAG
    -> Response: AI answer + product hits
    -> Frontend renders AI answer box above product grid
    -> Follow-up queries use conversation_id for context
```

### Components

#### 1. Schema Changes — Auto-Embedding Field

Add a vector field with `embed.from` to the product collection schema:

```json
{
  "name": "embedding",
  "type": "float[]",
  "embed": {
    "from": ["name", "description", "categories_text"],
    "model_config": {
      "model_name": "ts/all-MiniLM-L12-v2"
    }
  }
}
```

The `from` fields are configurable in admin. Typesense generates embeddings automatically using a built-in S-BERT model — no external API needed for embeddings.

#### 2. Admin Configuration

New section: **Conversational Search** (under TypeSense config)

| Setting | Type | Default | Description |
|---|---|---|---|
| Enable Conversational Search | boolean | No | Master toggle |
| OpenAI API Key | obscure | — | Encrypted, for LLM responses |
| OpenAI Model | select | gpt-4o-mini | Options: gpt-4o-mini, gpt-4o, gpt-4-turbo |
| System Prompt | textarea | (default prompt) | Customizable shopping assistant prompt |
| Embedding Source Fields | multiselect | name, description | Which indexed fields to embed |
| Conversation TTL | text | 86400 | Seconds before conversation history expires |

Default system prompt:
> You are a helpful shopping assistant. Answer questions about products based on the provided context. If you cannot answer from the context, say so politely. Include specific product names and prices when relevant.

#### 3. Backend — ConversationModelManager

New service that manages the Typesense conversation model lifecycle:

- On config save (when conversational search is enabled):
  1. Create conversation history collection if not exists
  2. Create or update conversation model with OpenAI config
- On config save (when disabled):
  1. Delete conversation model
- Stores the conversation model ID in config for frontend use

#### 4. Frontend — Enhanced Instant Search

When conversational search is enabled:

- The instant search JS switches from `search` to `multi_search` with `conversation=true` and `conversation_model_id`
- Renders an AI answer box above the product grid with the `conversation.answer` text
- Tracks `conversation_id` in session for follow-up queries
- Falls back to regular keyword search if conversational endpoint fails
- Product grid still shows the normal Typesense hits (same cards, facets, sorting)

#### 5. Indexer Changes

`ProductSchemaProvider` adds an `embedding` vector field with `embed.from` configured from admin settings. The embedding model config uses `ts/all-MiniLM-L12-v2` (built-in, no API key needed).

Changing embedding source fields in admin requires a full product reindex (same as changing additional attributes today).

### What This Does NOT Include (YAGNI)

- No separate chat page or modal
- No customer account conversation history
- No multi-collection RAG (products only for v1)
- No streaming responses
- No custom embedding models
- No non-OpenAI LLM providers
